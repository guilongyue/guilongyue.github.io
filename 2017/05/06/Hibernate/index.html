Hibernate 复习

  • ORM思想:Object Relation Mapping，对象关系映射，指的是Java对象和关系数据库之间的映射。

  • 主配置文件：通常为hibernate.cfg.xml.可以配置数据库连接参数，框架参数，以及映射关系文件。

一级缓存

Hibernate创建每个Session对象的时候，会给session对象分配一块独立的内存区域，存放session查询出来的对象。这个分配给session的缓存区称之为一级缓存。也叫Session级缓存。
一级缓存是默认开启的。
Session的save、update、delete操作会触发缓存更新。
session.evict(obj) 缓存中移除obj
session.clear() 清空缓存
session.close() 关闭session，释放缓存空间。

Hibernate中对象的状态

  • 临时态
    new delete的对象，可以被垃圾回收，不与session关联。
  • 持久态
    get load list iterate方法查询到的
    save update操作的
    垃圾回收不能回收，持久态的对象可以自动更新到数据库中，时机是在调用session.flush(),因此提交事务也会触发同步。
  • 游离态
    通过session的evict、clear 、close
    可以被垃圾回收，游离态的对象进行过持久化，单已与session解除了关联。

延迟加载

这种将查询时机推迟到对象访问时的机制称之为延迟加载。

  • session.load()
  • query.iterate()
  • 关联映射中对关联属性的加载。
  • 注意避免在使用对象前关闭session。query.get(),session.list()等。

Open session in view

在项目中，DAO只是负责查询出数据，而使用数据是在jsp解析的过程中，因而要避免在DAO中关闭session。或者说要在视图层保持session的开启。

  • 在servlet 过滤器
  • 在struts2 拦截器
  • 在Spring中使用AOP

延迟加载的实现原理

采用延迟加载方法，返回的独享是Hibernate采用CGLIB技术在内存中生成的类型，该类型是原实体类的子类，并在子类中重写了属性的get方法。

关联映射

若两张表有关联关系，我们可以在实体对象和映射关系文件中配置这种关系，然后使用Hibernate操作一张表示，可以自动操作到另一张表，这种通过配置自动操作另一张表的手段称之为关联映射。

一对多关联

  • 一方加集合属性
  • 一方映射文件中配置关联关系 Account.hbm.xml

多对一

一般情况下，多对一的关系中，多方丛属于一，员工从属于部门。因此很少根据多方数据增删改一方数据，更多的是对一方数据的关联查询。
Person.hbm.xml

关联操作

关联查询

  • 延迟加载
    默认情况下，关联属性是采用延迟加载机制进行加载的。
    lazy=“true”
  • 抓取策略
    fetch=”join” 连接查询，一起把对方数据查询出。此时延迟加载失效。
    fetch =”select” 不使用连接查询，默认的。

    级联操作

    cascade=”save-update”
    cascade =”delete”
    cascade=”all”

    控制反转

    在使用一对多，关联时，使用级联新增、删除时，当前操作的一方会试图维护关联字段，然而关联字段是在多方对象，他会自动维护这个字段，因此“一”方没必要做这样的处理。
    inverse = “true”，表示控制反转，及交出控制权
    默认inverse = “false”
