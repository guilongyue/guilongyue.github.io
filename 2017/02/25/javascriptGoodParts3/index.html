javascript good parts 对象

发表于 2017-02-25 | 分类于 笔记 |

对象

js中的简单类型包括数字、字符串、布尔值、undefined、null。其它谁有值都是对象，数字、字符串、和布尔值貌似对象，因为他们有方法，但他们是不可变的。js中对象是可变的键控集合，在js中，数组是对象，函数是对象，正则是对象，对象当然是对象。

对象是属性的容器，属性的名字可以使包括空字符串在内的任意字符串，属性值可以是除undefined外的任意值。
js中的对象是无类别的，他对新属性的名字和值没有约束，对象适用与收集和管理数据。对象可以包含其他对象，所以他们可以容易的展示成树形或图形结构。
js中包括一个原形链特性，允许对象继承另一对象的属性。正确的使用能减少对象初始化时间和内存消耗。

对象字面量

对象字面量提供一种非常方便的创建新对象值的表示法，一个对象字面量就是包在一对{}中的0个或多个名/值对，对象字面量可以出现在任何允许表达式出现的地方。
属性名可以使包括空字符串在内的任意字符串，在对象字面量中，如果属性名是合法的js标识符且不是保留字，并不强制要求用引号括住属性名。
逗号用来分隔名/值对。
对象的值可以从包括另一个对象字面量在内的任意表达式中获得。对象是可嵌套的。

检索

要检索对象中包含的值，可以采用在[]后缀中，括住一个字符串表达式的方式。如果字符串表达式是一个常数，而且他是一个合法的js表达式，而非保留字，那么可用.表示法代替。
如果你尝试检索一个并不存在的成员元素的值，将返回一个undefined。
||运算符可以用来填充默认值。
尝试检索一个undefined值将会导致TypeError异常，这可以用undefined来避免。

flight.equipment //undefined
flight.equipment.model //TypeError
flight.equipment && flight.equipment.model //undefined

更新

对象中的值可以通过赋值语句来更新，如果属性名以存在于对象中，那么这个属性的值被替换。
若对象没有这个属性名，那么该属性就被扩充到该对象中。

引用

对象通过引用来传递，他们永远不会被拷贝。

原型

每个对象都连接到一个原型对象，并且它可以从中继承属性，所有通过对象字面量创建的对象都连接到Object.prototype这个js中标准的对象。

当你创建新对象时，你可以选择摸个对象作为他的原型，js提供的实现机制杂乱而复杂，但其实可以被简化。我们将给Object加一个beget方法，这个beget方法创建一个使用原对象作为其原型的新对象。

原型连接在更新时是不起作用的，当我门对某个对象作出改变时，不会触及到该对象的原型。
原型连接只有在检索值的时候被用到，当我们尝试获取莫对象的属性值，且该对象没改属性名，那么js会试着从原型对象中获取属性值，如果那个原型对象也没有该属性，那么再从他的原型对象中寻找，以此类推，直到该过程最后到达终点Object.prototype，如想要的属性完全不在原型链中，那么就是undefined值，这个过程称为委托。
原型关系是一种动态的关系，如果我们添加一个新的属性到原型中，该属性会立即对所有基于该原型创建的对象可见。

反射

检查对象并确定对象有什么属性是一件很容易的事，只要试着去检索该属性并验证，typeof操作符对确定属性类型有帮助。
请注意原型链中的任何属性也会产生一个值。

typeof flight.toString(); //function
typeof flight.constructor(); //function

有两个方法处理这些不需要的属性，第一检查并剔除函数值，一般来说做反射的目的是数据，你应该意识到一些值可能会是函数。
另一个方法是hasOwnProperty方法，如果对象拥有独有的属相，返回true，他不会检查原型链。

枚举

for in语句可用来遍历一个对象中的所有属性名，该枚举过程会列出所有的值包括函数和原型链中的属性，可以过滤。最为常用的是hasOwnProperty方法，以及用typeof排除函数。

属性名出现的顺序是不确定的，如果你想要确保属性以特定的顺序出现，最好的方法就是避免使用for in 语句。而是创建一个数组，其中以正确顺序包含属性名。

删除

delete运算符可以用来删除对象的属性，他会移除对象中确定包含的属性，不会影响到的原型链中的任何对象。
删除对象的属性可能会使原型链中的属性浮现出来。

减少全局变量污染

js可以很随意的定义那些可保存所有应用资源的全局变量，不行的是全局变量消弱了程序的灵活性。应该避免。
