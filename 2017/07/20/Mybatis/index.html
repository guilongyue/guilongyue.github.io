Mybatis 复习

MyBatis是一个支持普通SQL查询，存储过程和高级映射的优秀持久层框架。MyBatis消除了几乎所有的JDBC代码和参数的手工设置以及对结果集的检索封装。MyBatis可以使用简单的XML或注解用于配置和原始映射，将接口和Java的POJO（Plain Old Java Objects，普通的Java对象）映射成数据库中的记录。

Mybatis 体系架构

  • 加载配置
    配置有两种方式，一种是注解，一种是xml配置文件，Mybatis将sql的配置信息记载成MappedStatement对象（包括传入参数配置、sql语句、结果映射配置），并将其存储在内存中。
  • SQL解析
    当API接口层收到收到调用请求，会接受传入的SQL的ID和传入对象（可以是Map、javabean和原始类型，），Mybatis根据SQl的id找到对应的MappedStatement，然后根据传入的参数对象对MappedStatement进行解析。解析后可以得到最终要执行的sql和参数。
  • SQL执行
    将最终得到的sql和参数，拿到数据执行。得到操作数据库的结果。
  • 结果映射
    将操作数据库的结果按照映射的配置进行转换，可以转换成HashMap、JavaBean、基础类型，并将最终结果返回。

    Mybatis配置文件

  1. SqlMapConfig.xml 1个
    主配置文件，用于指定数据库参数和连接参数。

  2. SqlMap.xml n个
    映射定义文件，用于定义SQL语句和映射信息。

1、namespace必须与对应的接口全类名一致。
2、id必须与对应接口的某个对应的方法名一致即必须要和UserMapper.java接口中的方法同名。

框架API简介

  • SqlSessionFactoryBuilder
    该对象负责根据MyBatis配置文件SqlMapConfig.xml创建SqlSessionFactory
  • SqlSessionFactory
    Mybatis应用的核心。用于创建SQLSession

  • SqlSession 包含了所有执行sql操作的方法，用于执行已映射的sql语句。

Spring与Mybatis整合

mybatis-spring.jar 提供了几个API

  • SqlSessionFactoryBean
    • dataSource属性数据源
    • mapperLocations指定mapper文件存放的位置。
      为整合应用提供SqlSession对象。
  • MapperFactoryBean
    根据指定Mapper接口生成Bean实例。
  • MapperScannerConfigurer
    根据指定包批量扫描Mapper接口并生成实例。
    在定义MapperScannerConfigurer，是需要指定一个basePackage即可。basePackage用于指定mapper接口所在的包，在这个包及其子包下面的mapper接口，都将被搜索到，并把他们注册成一个个MapperFactoryBean。
    如果指定的包下，不完全是我们定义的Mapper接口，此时可用MapperScannerConfigurer的两个属性。一个是annotationClass,一个是markerInterface。
